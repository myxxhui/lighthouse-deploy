# 成本透视前端功能与数据验证清单

部署更新后，待 **backend 健康**（首次启动需等 ETL 全量回填完成，约数分钟至 2 小时内），在浏览器中按下列项自检。

## 1. 访问入口

- 前端：http://\<主机\>:3000/cost（或你配置的 `LIGHTHOUSE_FRONTEND_PORT`）
- 若为远程服务器，需端口转发或通过 Nginx 暴露 3000/8080

## 2. 时间范围与总成本

- [ ] 顶部时间范围按钮组完整展示：**本月、上月、这季度、上季度、今年、去年**
- [ ] 选择「本月」或「上月」后，**全环境总成本**有数值（非一直加载/报错）
- [ ] 自定义为**月份选择器**（起止月份），最多可选 **5 年（60 个月）**

## 3. 成本分解总和 = 总成本

- [ ] **成本分析**四个大类（计算资源、存储、网络、安全等）的**金额之和** = 顶部**总账单成本/全环境总成本**
- [ ] 若不等，说明后端 `scaleDomainBreakdownToTotal` 未生效或需重启 backend

## 4. 云产品成本明细 · 趋势大图（单产品）

- [ ] 打开「趋势图」开关，表格出现**趋势**列
- [ ] 点击某一行的**趋势图**（小图）→ 弹出**大图**
- [ ] 大图标题为 **「{产品码} 成本趋势」**，且曲线为该**单产品**的日/月趋势，而非总成本趋势
- [ ] 若标题为「成本趋势大图」且曲线为总成本，说明未从该行带入 `trendModalProductCode`

## 5. 环比列

- [ ] 打开「环比」开关，表格出现**环比**列
- [ ] 有上期数据的产品显示**百分比变化**（如 +10%、-5%），而非全部「新增」
- [ ] 仅当上期无数据时该产品显示「新增」为符合预期

## 6. 上季度 / 今年 / 去年

- [ ] 选择「上季度」「今年」「去年」时，**有数据展示**（非一直空/0）
- [ ] 依赖月原始表已回填近 5 年；若仍无数据，确认已跑过**全量回填**（或等待首次启动回填完成）

## 7. API 快速校验（backend 健康后）

```bash
# 总成本与成本分解（本月）
curl -s "http://localhost:8080/api/v1/cost/global?period=month" | jq '{total_cost, domain_sum: ([.domain_breakdown[].cost] | add)}'

# 检查 domain_sum 与 total_cost 是否一致（允许小数误差）
```

---

**说明**：backend 首次启动会执行「全量检查 → 必要时 5 年月数据回填 → 聚合 → 对账」，HTTP 服务在此之后才监听。可通过 `docker logs lighthouse-deploy_backend_1` 查看进度，出现 `billing full backfill: ...` 或 `pipeline completed` 后，再访问前端与 API。
